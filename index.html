Suspense para datos

Deja de usar estados isLoading manuales. Aprende la forma declarativa de manejar la espera.

isLoading, isFetching, isPending...
Estado esparcido
Imperativo y tedioso

El clásico isLoading

function Profile() {
  const [user, setUser] = useState(null);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    fetchUser().then(data => {
      setUser(data);
      setIsLoading(false);
    });
  }, []);

  if (isLoading) return <Spinner />;
  return <div>{user.name}</div>;
}

Problemas: lógica de carga mezclada, difícil de componer, múltiples estados en diferentes componentes.

React.lazy + Suspense
Suspense para código

El mismo patrón para datos

import { lazy, Suspense } from 'react';

const Profile = lazy(() => import('./Profile'));

function App() {
  return (
    <Suspense fallback={<Spinner />}>
      <Profile />
    </Suspense>
  );
}

Suspense ya existe para división de código. La idea se extiende a la carga de datos: declarar qué mostrar mientras esperas.

wrapPromise simplificado
Con Suspense

Sin isLoading en el componente

// resource.js
const resource = wrapPromise(fetchUser());

function Profile() {
  const user = resource.read(); // Lanza promesa si no está listo
  return <div>{user.name}</div>;
}

// App.js
<Suspense fallback={<Spinner />}>
  <Profile />
</Suspense>

Mecanismo: read() lanza la promesa. Suspense la captura y muestra el fallback.

Código más limpio
Ventajas

¿Por qué Suspense?

Declarativo: el componente solo se preocupa por los datos listos.

Componible: anida Suspense sin estados adicionales.

Menos boilerplate: olvídate de useEffect + isLoading.

Transiciones: evita loading flashes con useTransition.

Cuándo usarlo
Conclusión

Adopta Suspense progresivamente

Regla simple:
¿Ves un estado isLoading en tu componente?
→ Considera usar Suspense + un resource.
¿Múltiples componentes con su propio loading?
→ Envuélvelos en un solo Suspense.

Resumen: Suspense separa qué cargar de cómo mostrar la espera. Tu código se vuelve más predecible y fácil de mantener.
